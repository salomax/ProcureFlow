## Overview

Primitives are the foundational components that form the basis of all other UI elements. They are simple, focused, and highly reusable.

## Components in this category:

- **Button** - Interactive elements for actions
- **Input** - Text input fields
- **Checkbox** - Binary choice selection
- **Radio** - Single choice from multiple options
- **Switch** - Toggle between two states
- **Badge** - Small status indicators
- **Avatar** - User profile images
- **Tooltip** - Contextual help text
- **Icon** - Visual symbols and indicators

## Usage Guidelines

1. **Consistency** - Use the same primitive across similar contexts
2. **Accessibility** - All primitives include proper ARIA attributes
3. **Theming** - Primitives adapt to light/dark themes automatically
4. **Responsive** - Components scale appropriately on different screen sizes

## Best Practices

- Use semantic HTML elements when possible
- Provide clear labels and descriptions
- Test with keyboard navigation
- Ensure sufficient color contrast
- Use appropriate sizing for touch targets
